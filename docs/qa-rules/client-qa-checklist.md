# Client QA Checklist

This checklist is sourced from `C:\Users\ahmar\OneDrive\Desktop\QA checklist.md` and is loaded as part of every design QA run.

## Logo

- Logo should be clickable and link to the homepage.
- Logo should use an approved production image format: PNG, SVG, or WebP.
- Logo alt text should include the site name or company name.

## Navigation

- Navigation should be flexible and should not rely on fixed heights where padding is more appropriate.
- Navigation should not break at any resolution or during browser resizing.
- Every menu item should be a real link with hover and active states.
- Hover states should be prominent and should not rely only on underlines.
- Menu items and toggle icons should have correct clickable areas.
- Submenus should open properly from arrow icons or submenu text.
- All menu items should be linked to approved destinations.
- External links should open in a new tab.
- Dropdown navigation should remain visible inside the viewport.

## Social Links

- Social icons should be proper links.
- External social links should open in a new tab.
- Social icons should have hover states and sufficient spacing on wrapped rows.

## Contact Links

- Email addresses should use `mailto:` links.
- Phone numbers should use `tel:` links.
- Email and phone links should be clickable and have hover states.

## Search

- Search fields should have correct styling, padding, vertical alignment, and button behavior.
- Search icons should be clickable buttons with cursor pointer behavior.

## Typography, Color, And Content

- Headings should follow a correct `h1` to `h6` hierarchy.
- Fonts should match the design for family, size, color, weight, style, line height, and letter spacing.
- Text and background colors should match the design file.
- Content should match the approved mockup or design source.

## Forms

- Form fields should use appropriate input types.
- Required validation should prevent empty submissions.
- Textareas should have proper dimensions, padding, and should not resize horizontally.
- Select controls should provide meaningful options and placeholder text.
- Submit buttons should have cursor pointer and hover states.
- Error messages should match the input type, be red, left aligned, spaced correctly, and should not show bullets.

## Images

- Images should match the mockup/design file.
- Images should be responsive and optimized.
- Image width and height should match the design across desktop and mobile.

## Browser Compatibility

- Website should display consistently across latest Chrome, Firefox, Edge, and Safari.
- Horizontal scroll is a bug.

## Responsive

- Font sizes, spacing, and image sizes should adapt across screen sizes.
- Section top and bottom spacing should be balanced on desktop and mobile.
- Structure should not break on any screen.
- Android, iOS, iPhone, and iPad layouts should be checked.

## SEO And Metadata

- Meta tags should be in proper format.
- Title tag should be unique, relevant, and in proper format.
