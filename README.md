# Design QA Cockpit

Dashboard-driven QA system for comparing an approved Figma design file with a developed page URL, then turning the results into GitHub-ready issues.

## What This Repo Contains

- `dashboard.html` - the browser cockpit for QA runs, markdown rule visibility, report review, and GitHub issue filing.
- `src/run-design-qa.js` - the automated QA runner.
- `src/qa/security-preflight.js` - URL, auth, Figma, security header, content secret-pattern, and markdown file access checks.
- `docs/qa-rules/` - international UI/UX and client checklist rules used by the runner.
- `.github/ISSUE_TEMPLATE/qa-design-comparison.yml` - required Figma URL and developed page URL issue form.
- `.github/workflows/design-qa.yml` - manual GitHub Actions workflow for design QA execution.
- `.github/workflows/pages.yml` - GitHub Pages deployment for the dashboard.

## Local Usage

```bash
npm ci
npm run build
npm run preview
```

Open:

```text
http://127.0.0.1:4173/dashboard.html
```

Run the full design QA flow:

```bash
npm run qa:design
```

Run browser and accessibility checks:

```bash
npm run qa
```

## Required QA Inputs

The QA system is intentionally strict:

- Figma design URL is required.
- Developed page URL is required.
- Figma API extraction requires `FIGMA_TOKEN`.
- Protected targets can use `HTTP_AUTH_USERNAME` and `HTTP_AUTH_PASSWORD`.
- GitHub issue creation requires `CREATE_GITHUB_ISSUES=true` and a token with issue-write access.

Copy `.env.example` to `.env` for local runs.

## GitHub Actions

Use **Actions > Design QA > Run workflow** and provide:

- `target_url`
- `figma_url`
- `create_issues`

The workflow uploads the full `reports/design-qa/` artifact after every run.

## Dashboard Markdown Dataset

The dashboard reads `public/dashboard-markdown.json`, generated by:

```bash
node scripts/sync-dashboard-markdown.js
```

This dataset includes QA docs, checklist rules, latest reports, and generated issue markdown so a user can see exactly what the system is checking.

## Security Notes

Do not commit real tokens into `dashboard.html`, `.env`, report artifacts, or generated screenshots. Use GitHub Actions secrets for:

- `FIGMA_TOKEN`
- `QA_GITHUB_TOKEN`
- `OPENAI_API_KEY`
- `ANTHROPIC_API_KEY`
